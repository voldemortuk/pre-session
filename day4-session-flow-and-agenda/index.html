01

SubAgents — Focused Delegation

A focused worker with its own context, tools and model

  • A subagent is the model plus a narrow harness: its own context window, its own tool allowlist, its own model. Results summarise back to the main thread, so your context stays clean.
  • The YAML frontmatter is the lesson — tools is least privilege, model: haiku is intelligent model routing, memory: project is persistent learning, description is how Claude decides when to delegate.
🖥 Hands-on — you will ~15 min
  • Delegate to the code-reviewer subagent over userService.js (a pointless O(n²) isEmailTaken) and users.js (no validation, never 404s) — and read its findings.
  • Have the test-writer subagent add the missing userService tests, running in its own context and reporting back.
  • Watch delegation keep the main conversation small — the whole point of a subagent is that only the result comes back.
02

Agent Teams — True Parallel Execution

Many teammates, a shared task list, no single orchestrator

  • Each teammate is a separate Claude instance with its own context, coordinating through a shared task list and messaging each other directly — already enabled in .claude/settings.json via CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1.
  • When teams beat subagents and when they don’t: parallel exploration and cross-layer work vs. sequential, same-file, dependency-heavy tasks — and why teams cost more tokens.
🖥 Hands-on — you will ~25 min
  • Spin up a lead + teammates to hunt the seeded bug in tasks.js PATCH: fields copied with if (req.body[field]), so any falsy value (false, "", 0) is silently dropped — un-completing a task returns 200 but changes nothing.
  • Watch them avoid two red herrings — the convoluted-looking merge and the stale-looking cache that are both actually correct.
  • Land the fix with a regression test for completed: false — the exact coverage gap that let the bug ship — holding the “one file, one owner” rule the whole way.
03

Dynamic Workflows — Move the Plan into a Script

Who holds the plan: the conversation, or a rerunnable script

  • A dynamic workflow is a script Claude writes that fans out subagents deterministically and you can rerun. The plan lives in the script, not the chat — so it repeats.
  • Where it earns its keep: codebase audits, large scripted migrations, and cross-checked research where independent agents verify each other before results land.
🖥 Hands-on — you will ~15 min
  • Run a workflow that audits users.js and tasks.js for validation and 404 gaps, fanning out subagents and aggregating their findings.
  • Save it to .claude/workflows/ and rerun it — same script, same coverage, no re-explaining.
  • Bonus: plugins that extend Claude Code, and git worktrees so parallel agents never collide on the same files.
04

Economics of AI Development — the Live Cost Lab

Part 3 · what AI-assisted work actually costs to run — measured live, not asserted

  • Shipping fast is the cheap part; running it is where the money goes. In AI-assisted work, OpEx is now a token bill — and context is a line item, not a technical detail.
  • The three levers, made measurable in a live interactive lab: a tight payload (AGENTS.md + only the files the task touches) instead of dumping the repo, prompt caching for the stable parts, and model routing — heavy models for the hard thinking, cheap models for tests, review and CI.
🖥 Hands-on — the AI Dev Cost Lab ~30 min
  • Shivam demos it live, then you work through its four activities yourself: Token Visualizer (paste text, see per-model pricing — a 500-token AGENTS.md vs a 50,000-token repo dump for the same output), Model Routing Challenge (classify five real tasks across cost tiers — ~73% saved vs all-premium), Prompt Cache Calculator (~90% off repeated context, one line of code), and Your 30-Day OpEx (all three levers on your own workload — 70–85% off).
  • The scale line to keep: 67% on a toy repo is a rounding error — the same 67% on a 100,000-token monorepo is 67,000 tokens saved every single turn.
  • Take-home reference: the AI Cost Playbook — 8 self-paced modules + glossary that back the lab.
